Matriz de Confusión:¶

La matriz de confusin de un problema de clase n es una matrz nxnen la que las filas se nombran segn las clases reales y las columnas, segn las clases previstas por el modelo. Sirve para mostrar de forma explcita cundo una clase es confundida con otra

Accuracy: Es el porcentaje total de elementos clasificados correctamente.

In [12]:
# funcin de matroz de confusin normalizada

def conf_matrix_norm(Y_test, Y_pred): # va a comparar los resultados de las predicciones de la variable target vs la real del testset
    cnf_matrix_tra = confusion_matrix(Y_test, Y_pred,  normalize='true')
    conf_plot = sns.heatmap(cnf_matrix_tra, annot=True, cmap='Blues', fmt='g')
    bottom, top = conf_plot.get_ylim()
    conf_plot.set_ylim(bottom + 0.5, top - 0.5)
    conf_plot.set_title("Confusion matrix, with normalization")
    conf_plot.set_ylabel('True Label')
    conf_plot.set_xlabel('Predicted Label');

Curva ROC:¶

Una curva ROC (curva de caracterstica operativa del recepto) es un grfico que muestra el rendimiento de un modelo de clasificacin en todos los umbrales de clasificacin. Esta curva representa dos parmetros:

  • Tasa de verdaderos positivos
  • Tasa de falsos positivos

El AUC proporciona una medicin agregada del rendimiento en todos los umbrales de clasificacin posibles. Una forma de interpretar el AUC es como la probabilidad de que el modelo clasifique un ejemplo positivo aleatorio ms alto que un ejemplo negativo aleatorio.

Conclusiones:¶

El objetivo de este estudio era la estimacin de diferentes modelos a partir de la base de datos aportada. Hemos llevado a cabo un anlisis pormenorizado cada una de las variables realizando los ajustes pertinentes para proceder a la modelizacin. Este ha incluido un anlisis exploratorio e ingeniera de variables. A continuacin hemos definido y entrenado los diferentes modelos: modelo nulo, modelo logit, SVM, random forest y XGBOOST. Despus de analizar los indicadores ms relevantes de cada modelo, como la precisin, el f-1 o el nmero de falsos negativos, y teniendo en cuenta el conocimiento de negocio, hemos concluido que el modelo ptimo es el modelo random forest hiperparametrizado.

La principal causa de esta decisin es que el nmero de falsos negativos de dicho modelo es el ms reducido de todos, por lo que las entidades prestamistas estaran evitando rechazar crditos a buenos prestatarios, y por tanto aumentaran sus beneficios.

Adicionalmente, mediante la curva de cumulative gains, hemos obtenido que nuestro modelo es capaz de predecir el 80% de los impagos con aproximadamente el 25% de la muestra y cerca del 100% de los impagos con el 60% de la muestra.

Respecto a las variables seleccionadas, es destacable que "last_fico_range_high" explica casi el 50% respecto a las dems que explican mucho menos del modelo. Esta varibale hace referencia al rango superior al que pertenece el FICO (credit score) del prestario en el momento en el que se origina el prstamo ("The upper boundary range the borrowers FICO at loan origination belongs to"). Es por ello, que es una variable de tanta importancia, ya que el historial de los prestarios suele determinar sus futuros cumplimientos o incumplimientos de deuda. Prueba de ello, es que durante la elaboracin de este proyecto, eliminamos dicha variable para estudiar los modelos resultantes y obtuvimos que la capacidad predictiva disminua considerablemente.

Para concluir, nos gustara destacar la importancia y dificultad del proceso de limpieza y seleccin de variables, siendo esencial el conocimiento de negocio, tanto para estas tareas, como para la seleccin del modelo ptimo.
